# 博客写作语法规则

## 一、Frontmatter 规范

每篇文章顶部必须用 `---` 包裹元数据，**所有引号必须使用英文半角引号 `""`，禁止中文全角引号 `""`**。

### 完整示例

```yaml
---
title: "文章标题"
description: "文章摘要（可选）"
date: 2026-07-15
updated: 2026-07-16
tags: ["标签1","标签2","标签3"]
categories: ["分类1","子分类2"]
draft: false
---
```

### 字段说明

| 字段 | 必填 | 类型 | 说明 |
|------|------|------|------|
| `title` | [必填] | 字符串 | 文章标题，用英文双引号包裹 |
| `description` | [可选] | 字符串 | 文章摘要，用于 SEO 和列表展示 |
| `date` | [必填] | YYYY-MM-DD | 发布日期 |
| `updated` | [可选] | YYYY-MM-DD | 最后更新日期 |
| `tags` | [可选] | 字符串数组 | 标签列表，如 `["散记","心得"]` |
| `categories` | [可选] | 字符串数组 | 分类路径，如 `["技术","前端"]` 表示 技术 › 前端 |
| `draft` | [可选] | true/false | 是否为草稿，草稿不会出现在网站上 |

### 常见错误

| 错误写法 | 正确写法 | 后果 |
|----------|----------|------|
| `title: "标题"` | `title: "标题"` | URL 中出现乱码字符 |
| `tags: ["标签","心得"]` | `tags: ["标签","心得"]` | 标签页面 URL 异常 |
| 缺少 `categories` 字段 | 加上 `categories: ["某分类"]` | 文章不会出现在分类树中 |

## 二、段落与换行

### 段落分隔 → 用空行

段落之间**空一行**（两个回车），每段会生成独立的 `<p>` 标签，CSS 自动首行缩进。

```markdown
这是第一段。内容内容内容。

这是第二段。内容内容内容。
```

### 段内换行 → 用空行替代

| 写法 | HTML 结果 | 缩进效果 |
|------|-----------|----------|
| `段落一[回车][回车]段落二`（空行分隔） | 两个 `<p>` | 正确：每段首行都缩进 |
| `段落一  [回车]段落二`（行尾双空格） | 一个 `<p>` + `<br>` | 错误：只有第一行缩进 |

**禁止使用行尾双空格换行**，那会把你整篇文章塞进同一个段落。

## 三、常用 Markdown 语法

### 标题

```markdown
# 一级标题
## 二级标题
### 三级标题
```

### 列表

```markdown
- 无序列表项
- 无序列表项

1. 有序列表项
2. 有序列表项
```

### 链接与图片

```markdown
[链接文字](https://example.com)

![图片替代文字](/path/to/image.png)
```

### 行内代码与代码块

````markdown
行内 `code` 用反引号包裹。

```python
# 代码块用三个反引号
print("hello")
```
````

### 引用

```markdown
> 这是一段引用文字。
```

### 分割线

```markdown
---
```

### 强调

```markdown
**粗体**
*斜体*
~~删除线~~
```

## 四、发布流程

1. 把 `.md` 文章文件放入 `posts/` 文件夹
2. 确保 frontmatter 中**引号为英文半角**、**段落用空行分隔**
3. 运行命令：

```bash
python publish.py "提交说明"
```

脚本会自动：
- 解析 frontmatter 并补全缺失字段
- 将文章复制到 `src/content/blog/`
- 构建网站 → 提交 Git → 推送到 GitHub
- 1-2 分钟后刷新 `treefish.top` 即可看到更新

---

## 五、快速检查清单

发布前逐项确认：

- [ ] `title` 和 `tags` 中的引号都是英文半角 `""`，不是中文 `""`
- [ ] frontmatter 包含 `categories` 字段（如需分类）
- [ ] 段落之间有空行，没有行尾双空格
- [ ] `draft: false`（除非确实不想发布）
- [ ] `date` 格式为 `YYYY-MM-DD`
